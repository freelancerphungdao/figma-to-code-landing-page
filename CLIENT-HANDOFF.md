# Client Handoff Checklist

Use this checklist before closing any landing-page freelance project.

## 1. Source Delivery

- [ ] GitHub repository access shared with client
- [ ] Branch naming and commit history are understandable
- [ ] README includes setup and customization notes

## 2. Build and Deployment

- [ ] Production build passes (`pnpm build`)
- [ ] Live URL is delivered
- [ ] Environment variables documented (if any)
- [ ] Deployment platform and project access shared

## 3. Design and Content Verification

- [ ] Final implementation checked against Figma
- [ ] Copy/text reviewed and approved
- [ ] Images/icons optimized and licensed
- [ ] Mobile/tablet/desktop responsive checks completed

## 4. UX and Accessibility

- [ ] Keyboard navigation works on interactive elements
- [ ] Focus states are visible
- [ ] Form validation and feedback are clear
- [ ] Contrast is acceptable for main text and buttons

## 5. SEO and Sharing

- [ ] Page title and meta description set
- [ ] Open Graph and Twitter tags set
- [ ] Favicon and social preview image available
- [ ] Canonical URL set for production domain

## 6. Contact and Conversion

- [ ] Contact form connected to agreed endpoint/service
- [ ] Test inquiry submitted successfully
- [ ] Primary CTA destinations verified

## 7. Post-delivery Support

- [ ] Warranty/support window communicated (e.g. 7-14 days)
- [ ] Scope for extra requests documented
- [ ] Next iteration suggestions shared with client

## 8. Files Shared to Client

- [ ] Source code repository link
- [ ] Live production URL
- [ ] Optional staging URL
- [ ] Brand assets used in final build
- [ ] Short edit guide for text/image replacement
